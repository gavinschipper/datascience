Een blik op de recensies van hotels in Amsterdam¶

Nadine Allewijn (11027541), Thomas Dijkman (11026057) Andrea Pineda (10590501) & Gavin Schipper (11027592)

Vrijwel iedereen boekt tegenwoordig hotels online via de vele websites die hiervoor beschikbaar zijn. Op deze websites is het makkelijk om reviews achter te laten en ook te lezen hoe anderen mensen hun verblijf bij een bepaald hotel hebben ervaren. Deze reviews zeggen echter misschien nog wel veel meer. Dit is wat wij graag wilden onderzoeken.

  • Onderzoeksvraag: Wat voor conclusies kunnen uit recensies van hotels in Amsterdam worden getrokken?
    • Is er een verband tussen de lengte van de recensie en het cijfer dat wordt gegeven?
    • Kunnen we het sentiment van een recensie voorspellen aan de hand van de gebruikte woorden? Oftewel; positiviteit of negativiteit.
    • Zijn er verschillen in recensies tussen de Amsterdamse buurten?
  • Data is afkomstig van Booking.com. Het bevat 515K aan recensies en scores van bijna 1500 hotels in het wat luxere segment in Europa.

# Deze functie maakt van een string een wordcloud met de bijvoeglijke naamwoorden die voorkomen in de string

def adj_wordcloud(string):
    tokens = nltk.word_tokenize(string)
    tags = nltk.pos_tag(tokens)
    
    #Hier selecteren we alle bijvoeglijke naamwoorden
    adjectives = " ".join([word[0] for word in tags if word[1] == "JJ" and word[0] != 'i'])
    
    wordcloud = WordCloud(font_path='/Library/Fonts/Arial.ttf',
                      background_color='white',
                      colormap="Blues",
                      max_words=200,
                      max_font_size=40,
                      scale=3,
                      stopwords = set(stopwords.words('english'))
                      ).generate(adjectives)
    fig = plt.figure(1, figsize=(12, 12))

    plt.imshow(wordcloud)
    plt.axis("off")

# We trainen een Naive Bayes Classifier met de reviews die we hebben

import nltk.classify.util
from nltk.classify import NaiveBayesClassifier
 
def word_feats(text):
    tokenized = nltk.word_tokenize(text)
    return dict([(word, True) for word in tokenized])
 
negreviews = amsterdam['Negative Review']
posreviews = amsterdam['Positive Review']

negfeats = [(word_feats(f), 'neg') for f in negreviews]
posfeats = [(word_feats(f), 'pos') for f in posreviews]

negcutoff = int(len(negfeats)*3/4)
poscutoff = int(len(posfeats)*3/4)

trainfeats = negfeats[:negcutoff] + posfeats[:poscutoff]
testfeats = negfeats[negcutoff:] + posfeats[poscutoff:]
print(len(trainfeats), len(testfeats))
 
classifier = NaiveBayesClassifier.train(trainfeats)
print('accuracy:', nltk.classify.util.accuracy(classifier, testfeats))

# We berekenen het fout percentage. Hierbij nemen 6 als de grens tussen onvoldoende en voldoende

incorrect = 0

for index,row in amsterdam.iterrows():
    if float(row['Reviewer Score']) < 6.0 and row['Predicted'] == 'pos':
        incorrect += 1
    elif float(row['Reviewer Score']) >= 6.0 and row['Predicted'] == 'neg':
        incorrect += 1
        
print("Foutpercentage =",(incorrect/len(amsterdam)*100))

Foutpercentage = 36.30055580801901

In meer dan 60 procent van de gevallen wordt de review dus correct geclassificeerd wanneer we de classificaties vergelijken met het cijfer dat de reviewer gegeven heeft. Dit is een prima resultaat. Een computer kan dus wel degelijk getraind worden om het sentiment van een review te voorspellen.
